# Contributing

Thank you for your interest in contributing to AL-ATP-001.

---

## Contribution Categories

### âœ… Accepted

- **Protocol refinements** based on synthesis trial results
- **Operator feedback** on clarity, timing, or measurements
- **Troubleshooting additions** from real cooking attempts
- **Substrate alternatives** with PLU documentation
- **Regional procurement variations** (different cities/stores)
- **Error corrections** in measurements or procedures
- **New protocols** within scope (Korean nutritional synthesis)

### ğŸ”„ Under Review

- **Fusion protocols** (requires justification for cuisine blending)
- **Non-standard equipment** requirements
- **Protocols requiring specialized training**
- **Significant departures from OSINT-verified sources**

### âŒ Rejected

- **Non-metric units** â€” This is a metric-only system
- **Unquantified instructions** â€” "Season to taste" is not valid
- **Protocols outside scope** â€” AL-ATP-001 focuses on Korean synthesis
- **Removal of safety/QC checkpoints** â€” These are non-negotiable

---

## Submission Requirements

All protocol contributions must include:

| Requirement | Description |
|-------------|-------------|
| **Reagent List** | Complete, with designations (D, G, K, etc.) |
| **Substrate List** | With PLU codes where applicable |
| **Equipment List** | Referenced to [[Equipment & Materials]] |
| **Step-by-Step Procedure** | Numbered, quantified, no ambiguity |
| **QC Checkpoints** | Visual or tactile verification points |
| **Troubleshooting Matrix** | Common failures and corrective actions |
| **OSINT Verification** | Cross-reference to established sources |

> [!tip] The Core Principle
> If an operator who has never made this dish before follows your protocol, will they succeed?
> 
> If you can't answer "yes" confidently, the protocol needs more detail.

## Human Trial Feedback

The most valuable contributions come from actual synthesis attempts.

### What to Report

| Field | Example |
|-------|---------|
| **Protocol** | ATP-A |
| **Date** | 2024-12-15 |
| **Outcome** | Success / Partial Success / Failure |
| **Deviations** | "Substituted zucchini for yellow squash" |
| **Observations** | "Needed 5 extra minutes for potatoes" |
| **Flavor Notes** | "Slightly too salty, will reduce doenjang next time" |
| **Photos** | Optional but helpful |

### Where to Submit

For now, direct feedback to the Principal Investigator. A formal logging system is under development.

1. **Initial Review** â€” Does the submission include all requirements?

3. **Format Alignment** â€” Match existing documentation style
4. **Integration** â€” Add to protocol suite
5. **Human Trial** â€” Test when possible
6. **Publication** â€” Release to vault

---

## Documentation Standards

### Voice & Tone

- Clear and direct
- Encouraging, not intimidating
- Lab terminology where it adds clarity
- No assumptions about prior cooking knowledge
- Explain *why* when it helps understanding

### Formatting

- Proper markdown with YAML frontmatter
- Consistent callout usage (tip, warning, note, etc.)
- Tables for specifications and matrices
- Mermaid diagrams for complex flows
- Footer with Aeryeok Labs branding

### Measurements

- **Always metric** (mL, g, cm)
- Tablespoon/teaspoon equivalents acceptable as secondary
- Specific times (not "until done")
- Temperature references where applicable
